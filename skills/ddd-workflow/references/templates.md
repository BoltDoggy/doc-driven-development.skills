# 文档模板

## 需求文档模板

路径：`docs/requirements/core-features.md`

```markdown
# 功能名称 - 需求文档

## 问题与目标用户
<!-- 本项目/功能解决什么核心问题？谁是目标用户？请用一句话说清楚 -->

## P0 功能（必须具备）
<!-- 哪些功能是必须有的？按优先级排列 -->

## P1 功能（锦上添花）
<!-- 哪些功能优先级较低，但很有价值？ -->

## 明确不做
<!-- 哪些功能在这个版本中明确不做？为什么？ -->

## 验收标准
<!-- 如何验证功能实现完成？每条必须可验证 -->
```

## 设计文档模板

路径：`docs/design/overall-architecture.md`

```markdown
# 项目名称 - 设计文档

## 架构概述
<!-- 项目类型（Web API / CLI / 库 / 前端应用等）、主要技术栈、整体架构风格 -->

## 核心模块与接口
<!-- 列出核心模块及其职责、关键接口的输入和输出 -->

## 数据模型
<!-- 核心实体及其关系、数据库模型、DTO 结构等 -->

## 错误处理
<!-- 错误类型、返回格式、异常策略 -->

## 技术取舍与限制
<!-- 当前设计中的技术取舍、已知限制、后续优化方向 -->
```

## ADR 模板

路径：`docs/adr/YYYY-MM-DD-title.md`

```markdown
# ADR-NNN: 标题

## 日期
YYYY-MM-DD

## 状态
[提案中 / 已接受 / 已弃用 / 已取代]

## 背景
<!-- 为什么需要做这个决策？有哪些可选方案？ -->

## 决策
<!-- 选择了哪个方案？选择的理由是什么？ -->

## 后果
<!-- 这个决策带来什么正面和负面后果？需要做什么迁移工作？ -->
```
